# paper/

Source files for the companion paper.

## Layout

- `short_paper.html` -- self-contained HTML build of the paper (single file with embedded CSS).
- `figs/` -- final figures referenced by the paper, in PNG and SVG.
- `figs/leads/` -- per-lead structural cards (if present in the rendered paper).

## Render to PDF

The HTML is print-ready. To produce a PDF:

```bash
# Headless Chrome (recommended; preserves vector SVG):
chrome --headless --print-to-pdf=paper.pdf paper/short_paper.html

# wkhtmltopdf alternative:
wkhtmltopdf --enable-local-file-access paper/short_paper.html paper.pdf
```

## Render to Word (journal submission)

Two Word files are produced for every submission, never one combined file:

- `short_paper.docx` -- main article (sections 1-7 + References).
- `short_paper_SI.docx` -- Supplementary Information (Appendix A-F).

Both use the `html2doc` skill's `review-manuscript` profile (Times New Roman,
1.5x line spacing, continuous line numbers, native editable Word equations).
See `../CLAUDE.md` for the full build recipe and the convention's rationale.

```bash
cd paper
python _split_html.py    # produces _body.html + _supplementary.html
# then run the html2doc 3-stage pipeline on each (see ../CLAUDE.md)
```

The `.docx` outputs are gitignored; rebuild them on demand from the committed
`short_paper.html`.
